01 Núcleo del Sistema

El backend está construido sobre Express.js optimizado para Vercel. Utiliza un motor de renderizado que intercepta las peticiones de archivos estáticos para inyectar metadatos.

Tenant Detection

El sistema identifica a la empresa mediante el req.headers.host (subdominios) o el parámetro slug en la URL. Esto permite que una sola instancia del servidor atienda a miles de marcas.

Bulk Processing

Utiliza multer y xlsx para procesar archivos en memoria. Las inserciones en base de datos se realizan mediante insertMany con ordered: false para resiliencia.

02 Flujo de Autenticación

1

Login & Token Issuance

Al validar credenciales, el servidor emite un AccessToken (15m) en el body y un RefreshToken (7d) en una cookie HttpOnly con flag Lax y Secure.

2

Silent Refresh

El frontend detecta la expiración del token localmente y solicita una renovación a /api/refresh-token. El navegador adjunta la cookie automáticamente, protegiéndola contra robos por JavaScript.

3

DDoS Protection

Middleware express-rate-limit configurado con ventanas de 15 min para APIs generales y límites estrictos para creación de pedidos por IP.

04 Comunicación en Tiempo Real

Broadcast Channels

Utilizamos Supabase Realtime para evitar el overhead de un servidor de Sockets persistente. El backend emite notificaciones a canales aislados por empresa.

Latencia < 100ms

Canal Global SuperAdmin

El SuperAdmin está suscrito al canal admin-global, recibiendo el stream de datos consolidado de toda la plataforma para monitoreo macro.
